All of these are along the process of learning to make the Connect 4 recreation

JavaScript
...

  1. DOM Manipulation:
  • The code interacts with the Document Object Model (DOM) to manipulate HTML elements on the webpage.
  • Elements are selected using document.querySelector and document.getElementById.
  • Event listeners are used to capture click and mouseover events.
  1. Event Handling:
  • The code utilizes the addEventListener method to handle click events on the document.
  • The click event triggers the main game logic.
  1. Game Logic:
  • The game involves a Connect Four grid represented by a 2D array (cells).
  • Players take turns clicking on cells, and the game checks for a Connect Four pattern horizontally, vertically, or diagonally.
  • The game maintains the state of the current player (currentPlayer) and alternates between players.
  1. CSS Style Modification:
  • The code dynamically modifies the CSS styles of elements, changing the color of player discs and updating the current player's color.
  • The --currentPlayerColor CSS variable is used to set the current player's color.
  1. Functions:
  • Several functions are defined to perform specific tasks, such as checking for a Connect Four, handling a win, refreshing the page, and finding the lowest available position in a column.
  1. HTML Structure:
  • The HTML structure includes a game board (gameBoard), hidden cells above the board for preview (aboveGameBoard), and individual cells representing the Connect Four grid.
  1. Page Reload:
  • A button is provided to reload the page, allowing players to start a new game.
  1. Loops:
  • The code uses nested loops for iterating through the game board cells and checking for patterns in different directions.
  1. Dynamic Element Creation:
  • Elements such as divs are dynamically created and added to the DOM within loops.
  1. Conditional Statements:
  • Conditional statements are used to determine the winner based on Connect Four patterns and to switch between players.

CSS
...

  1. CSS Variables: Defined custom properties using the --variableName syntax, such as --currentPlayerColor, --player1Color, etc., to store color values.

  2. Font Styling: Applied font styles to the body element using font-family, and set the font size and weight for .cell and .hiddenCell classes.

  3. Background Styling: Set background colors for body, .container, and .aboveContainer elements. Utilized RGBA notation for a transparent background (#ffffff00).

  4. Flexbox: Used Flexbox for layout, centering the content both horizontally and vertically within the body element.

  5. Grid Layout: Applied a grid layout to the .container and .aboveContainer elements with specified column configurations using grid-template-columns.

  6. Box Model: Adjusted padding and border-radius for .container, .aboveContainer, .cell, and .hiddenCell elements.

  7. Transition Effects: Employed transition effects on the .cell and .hiddenCell classes to smoothly animate changes in background color.

  8. Hover Effects: Added hover effects for elements with the .clickable and .clickable:hover classes, changing background colors on hover.

  9. Shadow and Filter Effects: Applied box-shadow to the .taken class, creating an inset shadow effect.

  10. Disc Customization: Customized player discs with background colors for .player1 and .player2 classes.

  11. Additional Styles: Mentioned the intention to add additional styles for game messages, etc., though not implemented in the provided code.
